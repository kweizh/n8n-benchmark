# n8n Edit Image Node Resize

## Background
n8n is a powerful workflow automation tool. You need to create a workflow that reads an image file from disk, resizes it using the Edit Image node, and writes it back to disk.

## Requirements
- The workflow must start with a Manual Trigger node.
- Read an image file from `/home/node/input.png` using the `n8n-nodes-base.readWriteFile` node (Operation: Read).
- The binary data property from the read node should be named `data`.
- Use the `n8n-nodes-base.editImage` node to resize the image.
- Set the width and height to 100x100 pixels.
- Set the resize option to "Maximum Area" (`maximumArea`).
- Write the resized image to `/home/node/output.png` using the `n8n-nodes-base.readWriteFile` node (Operation: Write).
- Save the workflow to `/home/node/workflow.json`.

## Implementation Guide
1. Create the workflow JSON structure with the required nodes.
2. The Manual Trigger node should have type `n8n-nodes-base.manualTrigger`.
3. The Read File node should have type `n8n-nodes-base.readWriteFile`, operation `read`, and `fileSelector` set to `/home/node/input.png`.
4. The Edit Image node should have type `n8n-nodes-base.editImage`, operation `resize`, `width` 100, `height` 100, and `resizeOption` `maximumArea`.
5. The Write File node should have type `n8n-nodes-base.readWriteFile`, operation `write`, `fileName` `/home/node/output.png`, and `dataPropertyName` `data`.
6. Connect the nodes in sequence: Manual Trigger -> Read File -> Edit Image -> Write File.
7. Save the JSON to `/home/node/workflow.json`.
